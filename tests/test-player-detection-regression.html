播放器誤判回歸測試

這個頁面混合了「真正播放器」與「看起來像播放器、但不應該被偵測」的元素，方便手動驗證無干擾播放器的 eligibility gating。

等待擴充功能掃描中...

Case 1: 真正 HTML5 播放器

這個應該被偵測，並可出現 `shield` 標記與無干擾播放器按鈕。

預期：應被偵測為播放器。

Case 2: 假播放器容器

class 名稱故意包含 player，但裡面沒有 video / iframe。

CLICK TO WATCH
預期：不應被偵測，不應出現任何圖示。

Case 3: Preview 縮圖影片

這是小尺寸、靜音、自動播放的預覽縮圖，不應進入候選名單。

預期：不應被偵測，不應被快捷鍵當主播放器。

Case 4: 廣告 iframe

容器與來源都帶有 ad / banner 訊號，應被排除。

預期：不應被偵測，不應進 popup list。

Case 5: Trap 影片

中等尺寸，但為 muted + autoplay + loop + no controls 的陷阱型態。

預期：不應被視為主要播放器。
